# Skill: Product Manager Role

> Role-specific emphasis loaded by the agent when the user selects this role.
> Applies to: Product Manager, Senior PM, Group PM, Technical PM, Growth PM,
> Platform PM, Product Lead.

---

## Target Role

Product Manager (4+ years senior-grade)

Variants this role covers:
- Product Manager · Senior PM · Group PM · Lead PM
- Technical PM · Platform PM · API PM · Developer-Tools PM
- Growth PM · Monetization PM · Conversion PM
- Product Marketing Manager (PMM) — adjacent
- AI / ML PM (emerging category — emphasize ML literacy)

---

## What to Emphasize

- **Launches with quantified outcomes**: features shipped + measured
  impact (DAU, retention, NPS, revenue, conversion). For senior PM,
  this is the single most important signal.
- **Cross-functional leadership**: how many engineers / designers /
  data scientists the candidate worked with; how many teams's roadmaps
  the candidate influenced.
- **Discovery and decision rigor**: user research, customer
  interviews, RFCs / PRDs authored, hypothesis testing in product
  decisions.
- **OKR / KPI ownership**: which company-level or team-level metrics
  the candidate was directly accountable for.
- **Strategy depth at senior**: market analysis, build/buy/partner
  evaluations, competitive positioning, roadmap prioritization
  frameworks (RICE, ICE, Kano, Cost-of-Delay).
- **Product-led growth artifacts** (when applicable): activation
  funnel improvements, viral loops shipped, pricing tier launches.
- **Industry depth signals**: domain (fintech, healthtech, gaming,
  SaaS, devtools, marketplace) and credibility within it.

## Summary

Target shape (3-4 sentences, ~70-110 words):

1. **Sentence 1** — Senior framing + years + dominant product
   surface (B2B SaaS / consumer / marketplace / fintech / etc.).
2. **Sentence 2** — Marquee launch with a quantified outcome
   (only if input has the number). E.g., "Shipped the redesigned
   onboarding flow that lifted W4 retention by 12 pts."
3. **Sentence 3** — Cross-functional scope or strategic strength
   (e.g., "Owned the activation funnel across N markets").
4. **Optional 4** — Industry credibility marker (talk, paper,
   patent, distinguished award).

Avoid:
- "Product manager with passion for building products"
- "Customer-obsessed"
- "Strategic thinker"
- "Drives results" (too generic)
- "Cross-functional collaborator" (everyone says this — show it)

Prefer:
- Specific surface owned ("the activation funnel")
- Specific cross-functional scope ("partnered with 3 eng teams +
  1 design team + 2 DS teams")
- Specific outcome with number when input has one

## Experience Bullets

Per-bullet template:

```
**Launch / Initiative Name** — *italic em-dash scope phrase if input
supports one.* [Verb] [WHAT product decision / launch]. [HOW with
discovery + execution]; [outcome with KPI movement ONLY if from input].
```

Concrete examples to mirror:

```
Onboarding Redesign — the activation funnel for new freemium signups.
Owned the rewrite from a 7-step legacy flow to a 3-step progressive
disclosure design; ran 4 A/B tests over 8 weeks across 3 markets.
Lifted W4 retention by 12 pts and cut time-to-first-value by 40%.
```

```
Pricing Tier Launch — the new mid-market plan introduced for teams
of 10-50. Authored the PRD, partnered with 2 eng teams + the data
science team on price-elasticity analysis, and managed the GTM with
Marketing. ARR contribution: $4.2M in the first two quarters.
```

What earns full marks:
- Action verb leads (Owned / Shipped / Designed / Launched)
- Surface scope is specific
- Cross-functional partners are named
- Outcome uses a real KPI movement (only if input has it)
- 3 lines max

What loses marks:
- "Worked on improving the onboarding flow"
- "Helped drive growth"
- "Managed stakeholders" (too vague)
- Listing every meeting attended

## Skills

Recommended skill bucket order (6-8 buckets):

1. **Product Discovery** — User research, customer interviews,
   problem framing, hypothesis design, jobs-to-be-done
2. **Strategy & Prioritization** — Roadmapping, RICE / ICE / Kano,
   cost-of-delay, opportunity sizing, build/buy/partner
3. **Execution & Delivery** — PRDs, design partnership, sprint
   collaboration, GTM coordination, launch playbooks
4. **Experimentation & Measurement** — A/B testing, holdout design,
   funnel / cohort / retention analysis, NPS / CSAT
5. **Analytics & Tooling** — SQL, Mixpanel, Amplitude, Heap, Tableau,
   Looker, dbt-light reading
6. **Cross-Functional** — engineering partnership, design partnership,
   data-science partnership, sales / CS interface
7. **Domain Depth** *(adjust per candidate)* — fintech / healthtech /
   marketplace / SaaS / DevTools / consumer
8. **PM Tools** — Jira, Linear, ProductBoard, Aha!, Figma, Notion,
   Confluence

Rules for the skills section:
- Lead with discovery and prioritization, not with tools
- Don't bury the analytical / SQL skills; senior PMs read data
- Skip "leadership" / "communication" buckets — show, don't list

## Education

- MBA from a recognized program is a positive signal at senior level
  (named program e.g., HBS, Wharton, Booth, Kellogg, INSEAD, ISB)
- B.S. / B.E. in CS / Engineering for Technical PM credibility
- Specialization (Strategy, Marketing, Operations) worth a one-line
  coursework note when relevant
- **No percentages / CGPA** for senior 4+ year resumes

## Achievements

High-signal items:
- Patents
- Conference talks (Mind The Product, ProductCon, BoS)
- Published articles (HBR, First Round, Reforge)
- Industry awards (Built In Best Places, recognized launches)
- Open-source product specs / templates with traction

Low-signal items (drop):
- Coursera / Reforge course completions (unless lead instructor)
- "Best Student" from undergrad
- Sports / cultural wins
- Generic "Top performer" awards without specifics

## Common Anti-Patterns for Product Manager

- Confusing PM with Project Manager: bullets that read like
  "managed timelines and coordinated standups" — emphasize product
  decisions, not project execution
- Listing every framework ever read about (Lean, Agile, Scrum, SAFe,
  Design Thinking, JTBD, OKRs…) — pick a few you actually use
- "Strong product sense" / "User empathy" — discount language; show
  examples instead
- Inventing impact numbers — Recruiters in PM check Crunchbase and
  LinkedIn for quoted launch outcomes; only state numbers from input
- Tech-heavy bullets that don't mention users or business — for PM,
  the bullet must close the loop to the customer or the metric
- Listing tools as the main skill ("Jira, Confluence, Mixpanel") —
  these are table stakes, the differentiator is what you used them
  for
