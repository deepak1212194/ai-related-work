# Role profile: AI / ML Engineer

For senior IC roles at AI labs, applied-ML teams at top-tier product
companies, and AI platform / infra teams.

## priority_keywords

LLM, RAG, retrieval-augmented generation, agentic AI, multi-agent, fine-tuning,
LoRA, QLoRA, instruction tuning, prompt engineering, embeddings, semantic
search, vector database, FAISS, hybrid search, hallucination mitigation,
evaluation, eval harness, guardrails, observability, model monitoring,
PyTorch, Hugging Face, Transformers, Sentence-BERT, SBERT, BGE, mpnet,
Llama, Qwen, Mistral, GPT-4, Claude, Azure OpenAI, Vertex AI, AWS Bedrock,
LangChain, LlamaIndex, CrewAI, Triton, TensorRT, ONNX, FP8, FP16, INT8,
NVIDIA, GPU, distributed training, FSDP, DeepSpeed, MLflow, Weights & Biases,
Azure ML, Vertex AI, SageMaker, Kubernetes, Docker, FastAPI, Python.

## hiring_signals

- A named, shipped system the candidate owned end-to-end.
- A choice the candidate made between two technical alternatives, with reason.
- A cost or latency improvement they personally measured.
- An evaluation method (held-out, A/B, offline metric) for a model they shipped.
- Patent / paper / talk counts for senior+ roles.

## red_flags

- Bullets that name only the framework ("Used PyTorch") without the system.
- Conflating prompt-engineering hobby work with production LLM ownership.
- Listing every cloud + every model without a coherent stack story.
- "10+ years experience" without a single named outcome.

## seniority_calibration

- IC4-IC5 (senior): expects 1-2 owned systems, real cost/latency numbers.
- IC6 (staff): expects multi-system architecture, cross-team influence,
  patent / publication, vendor or hardware-level decisions.
- IC7 (principal): expects org-wide impact, framework-level contributions,
  external speaker presence.
