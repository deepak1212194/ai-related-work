# Role profile: DevOps / Cloud / Platform Engineer

For SRE, platform, and DevOps roles at infra-heavy product companies.

## priority_keywords

Kubernetes, EKS, AKS, GKE, Helm, Kustomize, ArgoCD, Flux, Terraform,
Pulumi, CloudFormation, Ansible, AWS, GCP, Azure, IAM, VPC, networking,
service mesh, Istio, Linkerd, Envoy, observability, OpenTelemetry,
Prometheus, Grafana, Loki, Tempo, Datadog, New Relic, ELK, on-call,
incident response, blameless postmortem, SLO, SLI, error budget, runbook,
chaos engineering, GitOps, CI/CD, GitHub Actions, GitLab CI, Jenkins,
artifact registry, Vault, secrets management, Sealed Secrets, KMS, KEDA,
HPA, VPA, autoscaling, rightsizing, FinOps, cost optimization, blue-green,
canary, progressive delivery, zero-downtime, FedRAMP, SOC 2, compliance.

## hiring_signals

- A specific cluster size / workload count / region count.
- A CI/CD pipeline they architected with cycle-time numbers.
- An incident with MTTR / MTTD numbers.
- A cost-reduction initiative with $ saved or % reduction.
- IaC adoption: greenfield Terraform module, monorepo strategy.

## red_flags

- "Deployed code" with no platform / pipeline detail.
- Cloud bingo without a coherent stack story.
- No on-call mention for a role that explicitly expects on-call.
- Manual ops described as "automation".

## seniority_calibration

- Senior SRE: owned production system, on-call leader, runbook author.
- Staff: platform team lead, multi-region or multi-tenant design owner.
- Principal: org-wide platform strategy, vendor / framework decisions.
